Esquema de Datos

A continuación se presenta el esquema que siguen los datos que esta API provee, cabe destacar que este schema sigue todas las rúbricas establecidas por el Open Contracting Data Standard

Datos Globales

Con el objetivo de documentar el procedimiento de contratación, se realizan entregas periódicas de datos que se van generando durante las diferentes etapas del procedimiento de contratación. Una entrega es un conjunto de datos generados en un periodo especifico, a cada una de estas entregas se le asigna un identificador único utilizado para ligar un paquete de entregas global.

Campo Tipo de dato Descripción
ocid String! Identificador único del Proceso de Contratación Abierta. Este identificador fue asignado conforme a las rúbricas de la Open Contracting Identifier guidance
id String! Identificador único de la entrega/actualización de datos.
date String! Fecha en la cual esta información se publicó por primera vez.
tag [String] Uno o más valores de la lista de códigos releaseTag. Las etiquetas se pueden usar para filtrar las entregas y entender el tipo de información que puede contener una entrega.
title String Título descriptivo de la entrega o actualización de datos.
description String Descripción de la entrega o actualización de datos.
language String Especifica el idioma por defecto de los datos usando etiquetas de idioma de dos dígitos ISO639-1.
initiationType String! Cadena que especifica el tipo de proceso de inicio utilizado para este contrato, tomado de la lista de códigos de initiationType. Actualmente sólo se soporta el valor de "licitación" (tender).
parties [Organization] La información de las partes involucradas (organizaciones, operadores económicos y otros participantes) en el proceso de contrataciones y sus roles, por ejemplo: comprador, entidad contratante, proveedor, etc. Se utiliza referencias de organización en otras partes del esquema para hacer referencia a las entradas en esta lista.
planning Planning Información sobre la fase de planeación del proceso de contratación. Esto incluye información relacionada con el proceso de decidir qué contratar, cuándo y cómo.
tender Tender Objeto que contiene los datos relacionados a la etapa de licitación del procedimeinto de contratación.

Organizaciones

La información de las partes involucradas (organizaciones, operadores económicos y otros participantes) que están involucrados en el proceso de contrataciones y sus roles, por ejemplo: comprador, entidad contratante, proveedor, etc. Se utiliza referencias de organización en otras partes del esquema para hacer referencia a las entradas en esta lista.

Campo Tipo de dato Descripción
id [String] Identificador de la organización.
identifier Identifier Detalles especificos de la organización (RFC, nombre legal, esquema, etc.)
address Address Dirección de la organizacion
contactPoint ContactPoint Datos de contacto de la organización.
roles [String] Tipo de participación que tiene la organización en el procedimiento de contratación.
details [Details] Detalles adicionales de la organización.
shareholders [Shareholder] Datos de los accionistas que conforman la organización.

Planeación

Información sobre la fase de planeación del proceso de contratación. Esto incluye información relacionada con el proceso de decidir qué contratar, cuándo y cómo.

Campo Tipo de dato Descripción
rationale [String] Justificación de la contratación.
budget Budget Datos específicos del presupuesto.
documents [Documentation] Lista de documentos relacionados al proceso de licitación.
milestones [Milestone] Lista de hitos relacionados con la etapa de licitación.
project Project Esta sección puede contener información de proyectos relacionados con el procedimiento de contratación.
forecasts [Metric] Cualquier tipo de métrica o pronóstico establecido para el procedimiento de contratación.

Licitación

Esta etapa de la contratación se refiere al proceso de selección, en la cual se convoca a que distintos proveedores presenten ofertas. Este apartado está diseñado para almacenar información acorde al ti- po de procedimiento que se llevará a cabo. Durante la etapa de licitación se generan datos tales como la cantidad y las unidades en las que se comprarán los productos o servicios y también se establecen los requisitos que los concursantes deben cumplir para poder participar en las distintas etapas del con- curso.

Campo Tipo de dato Descripción
id String! Identificador para del proceso de Licitación..
title [String] Titulo del procedimiento de Licitación.
description [String] Resumen descriptivo del procedimiento de Licitación.
status [String] Estado del procedimiento de licitación al momento de registrar los datos.
procuringEntity ProcuringEntity Entidad convocante del Procedimiento de Licitación.
items [Item] Bienes y servicios por adquirir registrados durante el procedimiento de Licitacion.
value Value El valor tope estimado para la adquisición. Un valor negativo indica que el proceso de contratación puede implicar pagos del proveedor al comprador (comúnmente utilizado en los contratos de concesión)
minValue Value El valor minimo estimado para la adquisición. Un valor negativo indica que el proceso de contratación puede implicar pagos del proveedor al comprador (comúnmente utilizado en los contratos de concesión)
procurementMethod [String] Tipo de Licitación utilizado para el procedimiento de contratación
procurementMethodDetails [String] Detalles adicionales sobre el tipo de licitación utilizado.
procurementMethodRationale [String] Justificación para el tipo de Licitación utilizado.
mainProcurementCategory [String] Categoría principal que describe el objeto del procedimiento de contratación.
additionalProcurementCategories [String] Categorias adicionale que describen el objeto del procedimiento de contratación.
awardCriteria [String] Criterios de adjudicación para la adquisición award criteria codelist
awardCriteriaDetails [String] Detalles adicionales sobre el criterio de adjudicación.
submissionMethod [String] Especifica el metodo por el cual las ofertas deben ser presentadas, en persona, por escrito, o mediante licitacion electronica.
submissionMethodDetails [String] Mas informacion detallada sobre el metodo de presentacion de interes. Esto incluye la direccion, el correo electronico o el servicio en linea en el cual las ofertas deben ser presentadas.
tenderPeriod Period Detalles del periodo de Licitación. Esto incluye fecha de inicio, fin y duración total del período.
enquiryPeriod Period Información relacionada con el periodo de presentación de consultas.
hasEnquiries Boolean Un campo verdadero/falso para indicar si se recibió alguna consulta durante el proceso de licitación.
eligibilityCriteria [String] Una descripcion de cualquier criterio de eligibilidad para cualquier proveedor potencial.
awardPeriod Period Información acerca del periodo de adjudicación.
contractPeriod Period Informacion referente al período de contratación.
numberOfTenderers [integer] Número de licitantes.
tenderers [Organization] Lista de todas las organizaciones que fungen como Licitantes.
documents [Documentation] Lista de todos los documentos relacionados con el proceso de licitación.
milestones [Milestone] Lista de hitos asociados al proceso de licitación.
amendments [Amendment] Lista de modificaciones oficiales al proceso de licitación.
targets [Metric] Lista de métricas y previsiones relacionadas con el proceso de Licitación.
criteria [Criterion] Lista de criterios sobre los cuales los licitantes serán evaluados.
